Hi! My name is Josh Drentlaw, and welcome to Building Blocks: A Web Development Podcast. In this episode I'm going to introduce myself and talk a little bit about what this podcast is going to be about.

Currently, I'm not a professional web developer; I'm a warehouse worker in Southern California. I had an on-again, off-again relationship with web development for a few years, but recently decided I wanted to pursue web development full time. I'm currently building a few web sites in the hopes of getting a small freelance career off the ground. So far my clients have been great, and I'm really enjoying my journey.

When I started in web development, I did what probably so many people did: I learned just enough HTML and CSS to get working jQuery examples running, and that was it! Just like with most things, the fundamentals took a back seat to the more fun and (sometimes literally) flashy elements of web development. But now that I want to take this seriously, I find that I want to be able to do this right, not just for myself, but for people who it could really benefit.

What I'm talking about is web accessibility, or as it's known on the web, a11y. A11y is all about using special HTML tags and attributes that tell devices like screen readers what is happening on a web page. A <code>\<div></code> is really easy to reach for and use everywhere, but using tags like <code>\<main></code>, <code>\<section></code>, <code>\<nav></code>, and <code>\<article></code> give actual semantic meaning to your website. You can get by, and have a perfectly successful career never writing one shred of semantic HTML, but at some point, someone is going to come across your site and be sorely disappointed.
