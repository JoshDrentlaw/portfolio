Hi! My name is Josh Drentlaw, and welcome to Building Blocks: A Web Development Podcast. In this episode I'm going to introduce myself and talk a little bit about what this podcast is going to be about.

Currently, I'm not a professional web developer; I'm a warehouse worker in Southern California. I had an on-again, off-again relationship with web development for a few years, but recently decided I wanted to pursue web development full time. I'm currently building a few web sites in the hopes of getting a small freelance career off the ground. So far my clients have been great, and I'm really enjoying my journey.

When I started in web development, I did what probably so many people did: I learned just enough HTML and CSS to get a working jQuery example running, and that was it! Just like with most things, the fundamentals took a back seat to the more fun and (sometimes literally) flashy elements of web development. But now that I want to take this seriously, I find that I want to be able to do this right, not just for myself, but for people who it could really benefit.

What I'm talking about is web accessibility, or as it's known for the web, a11y. A11y is what's known as a numeronym, and it represents a way to distinguish web accessibility from accessibility in general. If you Google search accessibility you end up with a number of different topics that span that vast gamut of accessibility, so having a specific way to target and reference web accessibility is extremely handy.

Other famous numeronyms are 411, 911, and w3c. The 11 in a11y, besides reading ally in l33t speak, represents the 11 letters between a and y.

Accessibility is all about using special HTML tags and attributes that tell devices like screen readers what is happening on a web page. A <code>\<div></code> is really easy to reach for and use everywhere, but using tags like <code>\<main></code>, <code>\<section></code>, <code>\<nav></code>, and <code>\<article></code> give actual semantic meaning to your website. You can get by, and have a perfectly successful career never writing one shred of semantic HTML, but at some point, someone is going to come across your site and be sorely disappointed.

The reason I'm making this podcast is partially for myself I'll admit, but also for those of you, who like me, would like a way to learn good HTML and CSS fundamentals while you work your normal 9-5 job, whether that's in a warehouse, a coding job, or anything in between. These fundamentals are essential to creating a more compassionate internet, and we benefit as developers from learning the fundamentals.

How? Using a <code>\<div></code> tag in place of a <code>\<button></code> tag is talked about frequently on podcasts and I'm sure in some work places. If you know anything about HTML fundamentals you might be scratching your head wondering why on Earth anyone would ever do this, but apparently it does happen. People shouldn't feel shame for making a mistake like this, but knowing that <code>\<button></code> tags come with a lot of extra stuff built in that a <code>\<div></code> tag doesn't is a really handy thing to know.

I'm hoping your curiosity about this stuff is as great as mine, and I hope this podcast will help a new generation of developers make the web more accessible than it's ever been.

Thanks for listening, and I'll see you in the next episode. Bye!
